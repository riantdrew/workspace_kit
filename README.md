# Workspace Kit

**Память и правила для ИИ-ассистента в вашем проекте на Cursor.**

Версия kit: **1.13** · обновлено **19.06.2026**

---

## Зачем это вам

Каждый новый чат в Cursor начинается с чистого листа. Ассистент не помнит вчерашние договорённости, ваши приоритеты и то, что уже пробовали и от чего отказались.

**Workspace Kit** настраивает папку `workspace/` в вашем проекте: цели, правила, память команды и личный профиль. Ассистент читает их в начале работы и обновляет по итогам — вы не повторяете одно и то же каждый раз.

Этот репозиторий — сам kit: инструкция и шаблоны для внедрения в **ваш** проект за один разговор с ассистентом.

---

## Кому подходит

| Вы | Зачем workspace |
|----|-----------------|
| **Разработчик один** | ИИ помнит ваш стиль, стек и текущие задачи без длинных вводных в каждом чате |
| **Команда 2+ человек** | Общие цели и память в git; личные заметки — у каждого в своей папке |
| **Тимлид / продакт** | Видите приоритеты в `GOALS.md` и задачи в GitHub Issues; ассистент не уходит в сторону |
| **Новый участник** | Открывает `workspace/PROJECT.md` и `BOOTSTRAP.md` — быстрый вход в контекст |

Нужен [Cursor](https://cursor.com) и проект с кодом (или пустой репозиторий — тоже подойдёт).

---

## Что вы получите после внедрения

- **Предсказуемый ассистент** — знает роль, границы и то, что нельзя делать без спроса (деплой, коммиты, prod).
- **Память между чатами** — решения, уроки и ход работы не теряются.
- **Разделение «ваше / командное»** — общие файлы в git, личный дневник сессий не попадает в pull request коллег.
- **Задачи в GitHub Issues** — ассистент сверяет работу с доской, если вы попросите.
- **Обновления kit** — раз в неделю (или по команде) подтягиваются улучшения правил и навыков, без перезаписи ваших целей и личных данных.
- **Миграция при обновлении** — агент задаёт уточняющие вопросы, показывает diff-план и накатывает изменения безопасно, без жёстких скриптов.

Техническая инструкция для агента — в [`workspace_kit.md`](workspace_kit.md). Вам её читать не обязательно.

## Установка за 5 минут

### Шаг 1. Откройте чат в Cursor в своём проекте

Вставьте сообщение:

```
https://github.com/riantdrew/workspace_kit — помоги внедрить Workspace Kit
```

Агент загрузит актуальную инструкцию с GitHub, посмотрит ваш репозиторий и задаст несколько вопросов (стек, запреты, приоритеты на ближайшие недели).

### Шаг 2. Ответьте на вопросы и подтвердите создание файлов

Агент **не создаёт файлы без вашего согласия**. Сначала — короткий отчёт и план; после «да» — папка `workspace/` и настройки в `.cursor/`.

### Шаг 3. Начните рабочую сессию

В новом чате напишите:

```
@workspace/BOOTSTRAP.md
```

Ассистент прочитает цели, память и ваш профиль — и можно работать как обычно.

**Локальная копия kit (по желанию):** скачайте [`workspace_kit.md`](workspace_kit.md) в корень проекта и напишите в чате `@workspace_kit.md — помоги внедрить Workspace Kit`. Удобно без интернета; агент всё равно сверит версию с GitHub и предложит обновить, если на сервере новее.

## Как пользоваться каждый день

### Начало работы

Откройте чат и укажите bootstrap — или просто опишите задачу: на стадии 2+ ассистент сам подтянет `GOALS.md` и `MEMORY.md`.

**Примеры:**

```
@workspace/BOOTSTRAP.md — продолжаем рефакторинг модуля оплаты
```

```
Какие у нас приоритеты на эту неделю?
```

```
Покажи открытые задачи в GitHub
```

### Во время работы

Говорите с ассистентом как с коллегой. Папка `workspace/` подсказывает ему контекст — вам не нужно каждый раз копировать README.

**Примеры:**

```
Добавь endpoint по нашим конвенциям из workspace
```

```
Это противоречит GOALS? Если да — предложи альтернативу
```

```
Создай issue: «Добавить rate limit на /api/v1» — описание для команды
```

### Конец сессии

Напишите явно:

```
Завершаем сессию
```

Ассистент зафиксирует итоги в личном дневнике (`workspace/<ваш-login>/memory/`). Важные выводы для команды можно попросить перенести в `MEMORY.md`:

```
Зафиксируй в MEMORY: решили не трогать legacy API до релиза 1.5
```

## Что лежит в `workspace/` (краткая карта)

| Файл | Простыми словами |
|------|------------------|
| **BOOTSTRAP.md** | Ритуал начала и конца сессии — с чего ассистент стартует |
| **GOALS.md** | Приоритеты на период (2 недели — типично) и что **не** делаем сейчас |
| **MEMORY.md** | Проверенные факты и решения **команды** |
| **SOUL.md** | Кто ассистент в проекте: ценности, тон, границы |
| **PROJECT.md** | Кратко о продукте и стеке (без копирования всего README) |
| **CONSTITUTION.md** | Жёсткие правила: безопасность, архитектура, запреты |
| **AUTONOMY.md** | Что ассистент может сам, что — только с вашего «да» |
| **`<login>/USER.md`** | Ваш профиль: язык, детализация, экспертиза |
| **`<login>/memory/`** | Личный дневник по дням (не коммитится в git) |
| **knowledge/** | FAQ, runbook'и, справочники по домену |

Настройки Cursor (правила, хуки, команды) — в `.cursor/`, не в `workspace/`.

## Стадии внедрения — не всё сразу

Workspace Kit внедряется по шагам. Не нужно настраивать всё в первый день.

| Стадия | Что появляется | Когда имеет смысл |
|--------|----------------|-------------------|
| **1** | Безопасность: не коммитить секреты, не трогать prod без спроса | Сразу |
| **2** | Цели, память, профиль, bootstrap | После 1–2 ответов на вопросы агента |
| **3** | GitHub Issues, обмен с kit, навыки | Когда есть `gh` и задачи в Issues |
| **4** | Индекс, playbook, автожурнал сессий | Стабильная ежедневная работа с ИИ |
| **5** | Поддержка: актуальные GOALS, чистая MEMORY | Постоянный режим |

Агент на внедрении скажет, на какой вы стадии и что добавить дальше.

## Обновление kit

Kit на GitHub обновляется — новые правила, исправления, навыки.

**Если версия в проекте старее** — агент проведёт миграцию: задаст вопросы про git, layout, legacy-скрипты, покажет план изменений и применит только после вашего «да». Ваши цели, bootstrap, личные папки и секреты не трогаются без согласования.

**Вручную** — в чате Cursor:

```
/kit-exchange
```

или:

```
обнови kit / мигрируй workspace
```

**Автоматически** (рекомендуется со стадии 3): раз в неделю Cursor Automation подтягивает **только kit-managed** файлы в `.cursor/` и версию в `workspace/INDEX.md`. Ваши `GOALS`, `SOUL`, `BOOTSTRAP`, личные папки и секреты **не перезаписываются**.

Если в проекте лежит старый `workspace_kit.md`, агент при внедрении или обмене предложит обновить файл — согласитесь, если хотите локальную копию.

Канонический репозиторий: [github.com/riantdrew/workspace_kit](https://github.com/riantdrew/workspace_kit)

## Команда и личное

По умолчанию профиль **team**: общий репозиторий, 2+ человека.

- **Общее** (в git): `GOALS.md`, `MEMORY.md`, `SOUL.md`, knowledge, правила в `.cursor/`.
- **Личное**: `workspace/<ваш-github-login>/USER.md` — в git; `memory/` — только на вашем диске (в `.gitignore`).

Так коллеги не перетирают друг другу дневники, а цели и память команды — одни на всех.

Один разработчик: та же схема, Issues опциональны.

## Советы и лучшие практики

**Держите GOALS коротким.** 5–15 строк: три приоритета и явный out of scope. Детали задач — в GitHub Issues со ссылкой `#42` в GOALS.

**MEMORY — только проверенное.** Не догадки и не черновики. Устаревшее помечайте датой или удаляйте.

**Один факт — одно место.** Не копируйте README в workspace; в `PROJECT.md` — ссылка и 5 строк сути.

**Завершайте сессии фразой.** Тогда дневник заполняется сам; без этого контекст дня может потеряться.

**Не просите ассистента «запомнить в голове».** Попросите записать в MEMORY или создать issue — так это переживёт новый чат.

**Секреты — только в `.env`.** Не в workspace, не в чат. В `ENV.md` — имена переменных без значений.

**Проверяйте GOALS раз в 2 недели.** Скажите: «Сверь GOALS с открытыми Issues» — ассистент покажет расхождения.

**Обратная связь по kit** — через Issues в [репозитории workspace_kit](https://github.com/riantdrew/workspace_kit/issues). Или агент отправит обобщённый опыт при `/kit-exchange` (без ваших секретов и внутренних URL).

## Чего kit намеренно не делает

- **Фоновые задачи по расписанию** — кроме еженедельного обмена с kit (по вашей настройке).
- **Бесконечные циклы и автономный «агент 24/7»** — вы ведёте сессии сами.
- **Замена README и docs продукта** — workspace дополняет, не дублирует.

Так меньше сюрпризов: ассистент помогает в чате, а не живёт своей жизнью в репозитории.

## Частые вопросы

**Нужен ли git?**  
Желательно для команды и Issues. Без git workspace тоже работает — структура папок на диске.

**Обязательно ли класть `workspace_kit.md` в проект?**  
Нет. Достаточно ссылки на GitHub в чате.

**Ассистент создал файлы в корне репозитория — это нормально?**  
Нет. Контекст агента должен быть только в `workspace/`. Попросите перенести по инструкции kit.

**Можно ли править SOUL и CONSTITUTION?**  
Да, это ваши файлы. Ассистент меняет их только по вашей просьбе после первой настройки.

**Чем Workspace Kit отличается от User Rules в Cursor?**  
User Rules — глобально для всех проектов. Workspace Kit — про **этот** продукт: папка `workspace/`, цели, память команды, домен, запреты prod.

**Как узнать версию kit в проекте?**  
Откройте `workspace/INDEX.md` → `starter_kit_version`.

**Проект на старой версии kit — что делать?**  
Напишите в чате «обнови kit» или «мигрируй workspace». Агент задаст вопросы, покажет diff-план и применит изменения только после вашего согласия. Цели, bootstrap и личные данные не перезаписываются без подтверждения.
