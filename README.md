# AGENTS — инструкции для ИИ-агента в вашем проекте

**Версия шаблона:** 1.0

Каждый новый чат в Cursor начинается без памяти о прошлых разговорах. **AGENTS.md** — свод правил, который IDE подгружает в каждый диалог. Агент сам разворачивает `workspace/` и `.cursor/`, ведёт цели, память и отчёты. Вы задаёте стратегию — не рутину.

---

## Установка

1. Скопируйте [`agents.example.md`](agents.example.md) в корень **своего** проекта и переименуйте в `AGENTS.md`.
2. Откройте чат с любой задачей. Агент развернёт систему сам — отдельные команды не нужны.

Раздел «Проект» в AGENTS.md можно заполнить заранее или оставить пустым: агент задаст вопросы с вариантами ответа.

---

## Как это устроено

| Слой | Где | Что хранит |
|------|-----|------------|
| Законы | `AGENTS.md` в корне | Принципы, процедуры, роль агента |
| Активы | `workspace/` | Факты, цели, OKR, HADI, zettels, отчёты |
| IDE | `.cursor/` | Тонкие rules и hooks (создаёт агент при первом развёртывании) |

**AGENTS** — как думать и действовать. **workspace** — что известно о проекте сейчас. При обмене с другими отдавайте только AGENTS.md, не workspace.

---

## Что появится в вашем проекте

После первого разговора с агентом (молча, без ваших команд):

```
my-project/
├── AGENTS.md
├── .cursor/rules/          # указатели на AGENTS и workspace
├── .cursor/hooks/          # память сессий
└── workspace/
    ├── GOALS.md, MEMORY.md, USER.md
    ├── knowledge/, okr/, hadi/, metrics/
    ├── skills/, reports/
    └── memory/             # в .gitignore
```

## Гайд по разделам AGENTS.md

Ниже — подсказки при редактировании шаблона. В рабочем `AGENTS.md` этих блоков нет — только правила.

### Шапка и «Проект»

Универсальный шаблон. Применение:

1. Скопируйте или переименуйте в `AGENTS.md` в корне — полный файл, не указатель.
2. Заполните «Проект» и уберите ненужные разделы.
3. `AGENTS.md` — единственный источник ключевых подходов; `workspace/` — факты, цели, память.
4. Меняйте AGENTS только с явного разрешения. Агент предлагает правки сам.
5. `workspace/AGENTS.md` не создавайте — дублирование антипаттерн.
6. Факты и процедуры → workspace; подходы → AGENTS.
7. AGENTS можно делиться; workspace при обмене не отдавать.

**Что писать в «Проект»:**

- что за продукт или репозиторий и зачем;
- для кого (аудитория);
- главные цели и ограничения (стек, сроки, что нельзя ломать);
- карта каталогов — где код, конфиги, документация.

Публичный контекст — не секреты. Секреты, клиенты, внутренние URL — в workspace.

### Пример заполненного «Проект»

```markdown
## Проект

Веб-сервис подписок для Telegram-чатов. Аудитория — владельцы платных сообществ. Стек: PHP 8.3, Laravel, PostgreSQL. Код в `src/`, конфиги в `config/`. Нельзя ломать биллинг и существующие webhook-подписи. Приоритет — стабильность платежей и прозрачный статус подписки для пользователя.
```

### Душа агента

Роль (PM + аналитик + исполнитель; пользователь — стратегия), тон, уровень детализации, сторожевой режим по целям. Не дублируйте дословно «Обязанности» и «Стиль».

### Обязанности, вопросы, стиль

Правила поведения по одному на строку: сверка с памятью, отказ при риске, автозахват фактов, приоритеты. Стиль — инфостиль: суть в начале, «вы», один CTA, без канцелярита.

### Навыки

Повторяемые процедуры — в `workspace/skills/`; реестр — `workspace/SKILLS.md`. Пользователь не вызывает навыки командами.

### Инструменты

Только внешние интеграции (MCP, API). Базовые возможности Cursor не перечисляйте.

### Запреты

Действия без явного согласия и жёсткие лимиты на трату контекста.

### Политика

Роли, HADI/OKR/метрики, карта workspace, Zettelkasten, форматы, отчёты, gate, инфраструктура Cursor.

### Работа с сессией

Deploy, refresh, критик, старт, процесс, завершение.

## Примеры данных (создаёт агент в workspace/examples/)

### OKR (`okr/YYYY-QN.yaml`)

```yaml
period: "2026-Q2"
objectives:
  - id: O1
    title: "..."
    key_results:
      - id: KR1
        title: "..."
        target: 100
        current: 42
        unit: "%"
        status: at-risk
        confidence: medium
        metric_ids: [m_signups]
```

### HADI (`hadi/<id>.yaml`)

```yaml
id: hadi-001
title: "..."
hypothesis: "..."
action: "..."
data_sources: [github_issues, metrics/daily]
insight: ""
status: active
okr_links: [KR1]
opened: "2026-06-01"
closed: null
```

### Метрики (`metrics/schema.yaml`)

```yaml
metrics:
  - id: m_signups
    name: "Регистрации"
    unit: count
    cadence: daily
    source: api
    okr_link: KR1
```

### Zettel (`knowledge/zettels/zk-0042-auth-jwt.md`)

```yaml
---
id: zk-0042
title: "JWT срок 24 ч"
type: decision
status: stable
source: session-2026-06-22
created: 2026-06-22
updated: 2026-06-22
links: [zk-0010]
tags: [auth, security]
---
Токены доступа живут 24 ч; refresh — отдельный endpoint /auth/refresh.
```

### Фрагмент `knowledge/INDEX.md`

```markdown
## auth
- zk-0010 — модель ролей
- zk-0042 — JWT срок 24 ч

## audit
next_id: 43
zettel_count: 42
last_review: 2026-06-22
```

### `workspace/INDEX.md` (метаданные)

```text
agents_template_version: 1.0
agents_deployed: true
agents_deployed_at: 2026-06-22
agents_last_full_refresh: 2026-06-22
agents_refresh_interval_hours: 24
agents_last_session: 2026-06-22T14:30:00
```

## Обмен практиками

Универсальные улучшения из вашего AGENTS можно копировать в другие проекты вручную. Не передавайте workspace, секреты, внутренние URL и бизнес-метрики.

---

## Версия шаблона

В шапке `agents.example.md` и в `workspace/INDEX.md` поле `agents_template_version`. При обновлении шаблона сверьте версии и перенесите изменения в свой AGENTS.md вручную.
